Questão 2: Um cliente relata que está enfrentando lentidão em um determinado recurso do sistema. Como você abordaria esse problema para identificar a causa raiz e propor uma solução?
R: Quando um cliente relata lentidão em um recurso específico do sistema, para identificar e conseguir uma solução, preciso compreender a queixa do cliente, coletar dados de desempenho (por exemplo uma reunião para mostrar a tela) do sistema e isolar o problema. Após isso analiso a arquitetura do sistema para identificar possíveis causas de lentidão, incluindo verificações de consultas de banco de dados e verificação de logs em busca de anormalidades. Se não identificado nada, faço alguns testes para reproduzir o problema. 

Questão 3: Um usuário informa que está recebendo uma mensagem de erro ao tentar realizar uma operação específica no sistema. Explique como você investigaria esse problema e quais etapas seguiria para solucioná-lo.
R: Para analisar a mensagem de erro, é quase o mesmo processo de lentidão, começaria compreendendo a mensagem de erro reportada pelo usuário e tentaria replicar o erro em um ambiente de teste. Analisaria os logs do sistema em busca de informações sobre o erro. Verificaria o código-fonte relacionado à operação que causa o erro e traçaria um fluxo de execução para identificar a origem do problema. Com base na análise, corrigiria o código e testaria a solução.

Questão 4: Descreva como você lidaria com uma situação em que um usuário encontra um erro desconhecido.
R: Para um erro desconhecido, também é o mesmo fluxo de analise, coletaria informações sobre o erro com o cliente, tentaria replicar o erro em um ambiente de teste. Após replicar o erro no ambiente de teste, verificaria os logs do sistema para identificar o erro e posteriormente corrigi-lo.
